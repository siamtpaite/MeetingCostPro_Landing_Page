MeetingCost Privacy Policy

Last updated: April 7, 2026

MeetingCost is a browser extension that estimates meeting cost and optional PRO features (analytics, AI-assisted transcription, and calendar integration). This policy describes what data is processed, where it stays, and what you control.

Data stored on your device

By default, participant names, salary or rate assumptions, currency, templates, meeting history entries, and related settings are stored in chrome.storage.local in your browser profile. This data does not leave your device except where you explicitly enable features that call external services (below).

Meeting pages and metadata

On supported meeting sites, a content script reads page signals (for example URL patterns and visible controls) to detect when a call is active. It does not read passwords or unrelated site data. Overlay UI is injected only on meeting origins listed in the extension manifest.

Optional AI transcription and summarization

If you use PRO AI features, audio may be captured from your meeting tab or a source you choose, then sent to the third-party AI provider you select (for example AssemblyAI, OpenAI, or Deepgram) using an API key you supply. The API key is held in the extension service worker memory for your current browser session only and is not written to persistent extension storage. Transcripts and summaries are stored locally according to your retention settings unless you export them yourself.

Google Calendar (read-only)

If you connect Google Calendar, the extension requests read-only access to list upcoming events to show in the dashboard. OAuth tokens are handled through Chrome’s identity APIs and provider endpoints; MeetingCost does not sell this data.

Payments and licensing

Purchases may be processed by Gumroad, Plisio, or other providers you select. Those processors receive the data required to complete payment (for example email for delivery). License verification may call a configured HTTPS worker or Gumroad’s API. See each processor’s privacy policy for how they handle payment data.

Firebase and cloud backup (optional)

Optional cloud signup uses Firebase Authentication REST endpoints from the extension. Session tokens needed for refresh are stored locally in chrome.storage.local under the Firebase auth key. No remote Firebase JavaScript SDK is loaded from CDN in the shipped extension.

Retention and deletion

  • Clear meeting history, templates, or AI results from the extension UI or uninstall the extension to remove local data.
  • AI API keys in memory are cleared when the service worker restarts; use “Apply AI settings” again after a long idle period if needed.

Contact

For privacy questions, contact the publisher email shown on the Chrome Web Store listing.
